
### 表1：分维度回归模型配置表 (Master Regression Configuration)

**核心方程**：$Y_{i, t+1} = \alpha + \beta \cdot \text{Score}_{i,t} + \gamma \cdot \text{Controls}_{i,t} + \text{FixedEffects} + \epsilon_{i,t}$
*(注意：自变量是 t 年，因变量是 t+1 年)*

| 研究维度 (Dimension)                   | 核心自变量 ($X_{i,t}$) <br> *(需 Z-score 标准化)*                                      | 核心因变量 ($Y_{i, t+1}$) <br> *(预测目标)*                      | 预期符号 ($\beta$)                     | 经济学含义 (Story)                                         |
| :--------------------------------- | :---------------------------------------------------------------------------- | :------------------------------------------------------ | :--------------------------------- | :---------------------------------------------------- |
| **I. 技能** <br> (Skill)             | 1. **Logical Density** <br> 2. **Info Granularity** <br> 3. **Uniqueness**    | **$Alpha_{t+1}$** <br> (FF3 / FF5 Alpha)                | **正 (+)**                          | 逻辑清晰、信息颗粒度高、观点独特的经理，拥有真实选股能力，能持续跑赢市场。                 |
| **II. 行为偏差** <br> (Bias)           | 4. **Self-Serving** <br> 5. **Overconfidence**                                | **$Alpha_{t+1}$** <br> *或者* <br> **$Volatility_{t+1}$** | **负 (-)** <br> *或者* <br> **正 (+)** | 甩锅或过度自信的经理，往往伴随着未来的业绩下滑（Reversal）或更高的非理性波动风险。         |
| **III. 代理问题** <br> (Agency)        | 7. **Obfuscation** <br> 8. **Accountability** (负向) <br> 9. **Focus Shifting** | **$NetFlow_{t+1}$** <br> (资金净流入)                        | **负 (-)**                          | 投资者是聪明的（Smart Money）。如果经理故意模糊信息或转移话题，投资者会用脚投票，导致资金流出。 |
| **III. 代理问题 (进阶)** <br> *(JBF 同款)* | **$Perf_t \times Agency\_Score_t$** <br> (交互项)                                | **$NetFlow_{t+1}$**                                     | **负 (-)**                          | **流量-业绩敏感度 (FPS)** 降低。当代理问题严重时，即使业绩好，投资者也不买账（信任折扣）。   |

### 表2：控制变量清单 (Control Variables Checklist)

**所有控制变量均为 $t$ 期数据**。除收益率外，建议对连续变量进行 1% Winsorize 处理。

| 变量代码           | 变量名称      | 计算方式/定义                                                   | 预期符号 (对 Alpha/Flow)  |
| :------------- | :-------- | :-------------------------------------------------------- | :------------------- |
| **Past_Perf**  | **当期业绩**  | $Ret_{i,t}$ 或 $Alpha_{i,t}$。**这是最重要的控制变量**，用于剔除动量效应。      | + (强者恒强)             |
| **Log_TNA**    | **基金规模**  | $\ln(\text{Total Net Assets}_{i,t})$。大基金较难产生Alpha，但吸金能力强。 | - (Alpha) / + (Flow) |
| **Log_Age**    | **基金年龄**  | $\ln(1 + \text{Age in Years})$。老基金更稳健。                    | -                    |
| **Expense**    | **费率**    | (管理费+托管费)/TNA。费率高通常业绩差。                                   | -                    |
| **Turnover**   | **换手率**   | 年度股票交易额 / 平均净资产。                                          | +/- (不确定)            |
| **Volatility** | **历史波动率** | $t$ 年内日度/周度收益率的标准差。                                       | - (高风险低回报)           |
| **Log_Flow**   | **当期流量**  | $\ln(1 + Flow_{i,t})$。控制流量的自相关性。                          | +                    |
| **Inst_Hold**  | **机构持仓**  | (可选) 机构投资者持有比例。机构通常更理性。                                   | +                    |

### 表3：模型设定与稳健性检查 (Specifications)

| 设置项 | 具体要求 | 备注 (导师建议) |
| :--- | :--- | :--- |
| **回归方法** | **OLS 面板回归** | 使用 Stata 的 `reghdfe` 命令。 |
| **固定效应 (FE)** | **Year FE + Style FE** | **Year FE**: 吸收宏观冲击（必加）。<br> **Style FE**: 吸收投资风格差异（如成长/价值/行业主题）。 |
| **标准误 (SE)** | **Cluster by Fund ID** | `vce(cluster FundCode)`。解决同一基金在时间序列上的自相关。 |
| **标准化** | **Z-Score** | 对9个 LLM 指标做：$x' = (x - \mu) / \sigma$。方便解释系数大小。 |
| **样本筛选** | **Active Equity Funds** | 剔除指数基金、债券基金、保本基金。只保留主动权益类。 |

### 导师的执行指令 (Action Plan)

1.  **第一步 (Baseline)**：先跑 **单变量回归** (不加控制变量)，看 $Score$ 对 $Y$ 的符号是不是符合预期。
2.  **第二步 (Controls)**：加入所有控制变量，看系数是否依然显著。如果加了 Past Performance 就不显著了，说明你的文本指标只是业绩的“影子”，没有增量信息。
3.  **第三步 (Comparison)**：把你队友做的 **传统指标 (Traditional Metrics)** 放入同一个方程。
    *   模型：$Y = \alpha + \beta_1 \cdot \text{LLM\_Score} + \beta_2 \cdot \text{Traditional\_Score} + Controls$
    *   **目标**：看到 $\beta_1$ 显著，$\beta_2$ 不显著。
